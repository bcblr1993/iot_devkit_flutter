# IoT DevKit 产品级整改清单

更新时间：2026-05-12

## 验收原则

- 所有核心路径必须先有测试，再进入发布。
- 启动模拟前必须阻止明显错误、危险或会让用户误判状态的配置。
- UI 只展示一个明确状态来源，避免同一信息在多个区域重复或冲突。
- 运行中不允许出现“按钮显示运行，但实际没有设备/没有上送”的灰色状态。
- 所有用户可见文案必须走国际化，除协议名、单位、日志原文外不硬编码。

## 阶段 1：模拟配置校验

- [x] 新增统一配置校验器，覆盖 MQTT、基础模式、高级模式、自定义 Key。
- [x] 校验端口范围、Topic、设备范围、设备数量、发送间隔、数据点数量。
- [x] 校验分组名称、分组设备范围、全量/变化频率、变化比例。
- [x] 校验自定义 Key 空名称、重复名称、随机范围、静态值类型。
- [x] 启动前展示具体错误，而不是只提示“表单验证失败”。
- [x] 补充校验器单元测试。
- [x] 补充启动入口 widget 测试。

## 阶段 2：运行状态机

- [x] 将 `isRunning/isBusy/isStarting/isStopping` 收敛为明确状态模型。
- [x] 增加 `idle / starting / connecting / running / reconnecting / partialRunning / stopping / failed`。
- [x] MQTT 连接失败时展示可理解状态和失败原因。
- [x] 增加启动中停止、连接失败进入重连状态的测试。
- [x] 增加停止超时、部分连接成功的测试。

## 阶段 3：日志与监控信息架构

- [x] 底部日志条只负责日志入口、日志状态、最新日志预览。
- [x] 设备数、在线数、成功数、失败数、CPU、内存迁移到模拟器状态区。
- [ ] 日志展开态提供过滤、搜索、复制、清空、导出、自动滚动开关。
- [ ] 日志空态、过滤空态、错误态统一设计。
- [ ] 补充大量日志 UI 性能和窄窗口溢出测试。

## 阶段 4：动态表单稳定性

- [ ] 自定义 Key 表单从 `initialValue` 改成稳定 controller 或独立 row state。
- [ ] 分组表单从 `initialValue` 改成稳定 controller 或独立 row state。
- [ ] 导入配置、折叠展开、搜索过滤、切换主题后输入状态不丢失。
- [ ] 增加重复 Key、删除 Key、搜索后新增 Key 的 widget 测试。

## 阶段 5：反馈与国际化统一

- [ ] 新增统一反馈入口，替换零散 `SnackBar`、内联提示、临时 Toast。
- [ ] 所有用户可见文案进入 ARB。
- [ ] 统一成功、警告、错误、处理中反馈样式。
- [ ] 修复性能面板、模拟启动弹窗里的英文硬编码。

## 阶段 6：专业监控面板

- [ ] 监控面板增加运行状态、吞吐量、成功率、失败率、延迟、带宽。
- [ ] 空态不再显示粗糙硬编码文案。
- [ ] 图表支持平滑但不牺牲实时性。
- [ ] 大数量模拟下监控刷新频率受控，避免 UI 卡顿。

## 阶段 7：工时记录细节

- [ ] 小时输入限制合法小数格式，避免 `1.2.3`。
- [ ] 复制周报使用统一反馈组件。
- [ ] 删除确认和保存反馈在不同主题下视觉一致。
- [ ] 补充新增、编辑、删除、复制周报 widget 测试。

## 阶段 8：设置、主题、语言

- [ ] 主题选择和语言选择统一弹窗结构。
- [ ] 主题预览加入可读性检查，避免低对比度主题。
- [ ] 设置菜单在深色/浅色/窄窗口下不遮挡主内容。
- [ ] 补充主题切换、语言切换、关于弹窗测试。
