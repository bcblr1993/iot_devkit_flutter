## 新增

- 新增自动多进程模拟：按每个发送进程 300,000 点/秒的容量上限，在开始模拟前计算最坏峰值并自动启动所需工作进程。
- 新增设备连续分片和隐藏工作进程模式，主界面负责统一启动、停止、异常清理与统计汇总，避免设备范围、Client ID 和发送任务重复。
- 启动预览新增常态速率、全量峰值、单进程上限和所需进程数提示；超过机器安全进程数或设备无法继续分片时会阻止启动并明确提示。
- 新增 MQTT 容量基准工具和性能调优文档，支持按分片参数复现单进程与多进程发送上限。

## 修复

- 修复 5 分钟全量上送与秒级变化流叠加时，容量估算只看平均速率、导致单进程瞬时过载的问题。
- 分离调度器过期丢弃与真实 MQTT 发布失败统计，避免把未进入发送队列的过期 tick 误报为网络发送失败。
- 强化工作进程启动、停止和异常退出生命周期：任一进程启动失败或意外退出时会快速停止并清理全部进程，随后允许重新启动。
- 修复工作进程连续 IPC 状态输出可能丢失最终 `stopped` 事件的问题。

## 优化

- 优化高频遥测数据生成、UTF-8 编码缓冲、日志记录和调度热路径，降低高点数发送时的主 isolate 压力。
- 状态栏与性能监控支持汇总多个工作进程的在线数、吞吐、成功数、失败数和内存数据。
- Windows 工作进程以隐藏窗口方式运行，macOS 工作进程在关闭窗口后仍保持后台发送生命周期。
